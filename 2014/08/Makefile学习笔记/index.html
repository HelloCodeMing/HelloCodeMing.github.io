概述

make是linux下一个用于自动化编译的工具，能够处理依赖关系，自动进行编译、安装、测试等工作。

Makefile则是描述编译规则的文件，当我们运行“make”命令时，它会自动查找当前目录下的Makefile或makefile文件，按照文件描述的规则进行编译或安装等动作。

如何工作

  1. make首先会在当前目录查找名为Makefile或makefile的文件。
  2. 如果找到，它会在文件中寻找目标（target），在上面的例子中，target就是all。
  3. 如果target不存在，或者target后面的依赖文件修改时间比target要早，那么make就会先检查依赖文件，再执行target下面的命令。
  4. 在检查依赖文件的过程中，make会把依赖当做另外一个target继续进行以上的过程。
  5. 当所有的依赖都执行完之后，才会执行最上层target的命令

整个的执行过程会生成一棵树，根就是最终target，树叶就是每一个具体的源代码，所以Makefile只有一个入口，常用的入口有all、install、clean、test。

语法

书写规则也就是Makefile的语言，写Makefile就相当于写代码。

基本规则

targets是文件名，prerequisites是依赖文件，以空格分隔，command是执行的命令。command必须以tab开头。

###隐式规则
make可以自动推导target的依赖，比如说“main.o”就会自动产生依赖“main.c”，同时它的编译过程“cc -c main.c”也会被推导出来，我们就不需要把每一个编译的细节写出来了。
